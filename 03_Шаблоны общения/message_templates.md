# Шаблоны общения

Шаблоны используются как основа сообщения. Перед отправкой нужно адаптировать текст под конкретный бизнес и убрать все лишнее.

## Первое сообщение

Здравствуйте, [ИМЯ]. Посмотрел [БИЗНЕС] в нише [НИША]. Кажется, для вашей задачи [ЗАДАЧА] может быть полезна более понятная цифровая точка входа для заявки. Могу коротко показать, где сейчас клиенту может быть сложно оставить заявку. [СЛЕДУЮЩИЙ_ШАГ]?

## Ответ после интереса

[ИМЯ], спасибо за ответ. Чтобы не предлагать лишнее, уточню пару вещей по [БИЗНЕС]: какая сейчас главная задача - [ЗАДАЧА], заявки, каталог, запись или понятное описание услуг? После этого предложу самый короткий следующий шаг: [СЛЕДУЮЩИЙ_ШАГ].

## Вопрос про дизайн

[ИМЯ], по [БИЗНЕС] важно понять не только стиль, но и то, как клиент будет принимать решение. Есть ли у вас примеры визуального направления, которое нравится для [НИША]? Если нет, можем начать с простой структуры под [ЗАДАЧА], а затем согласовать дизайн. [СЛЕДУЮЩИЙ_ШАГ].

## Вопрос про бюджет

[ИМЯ], чтобы корректно сориентировать по бюджету, нужно понять объем: что именно должно быть внутри решения для [БИЗНЕС] и какая задача приоритетная - [ЗАДАЧА]. Можем сначала зафиксировать первый этап, а потом уже обсуждать стоимость. [СЛЕДУЮЩИЙ_ШАГ]?

## Сообщение после паузы

[ИМЯ], добрый день. Возвращаюсь к нашему разговору по [БИЗНЕС] и задаче [ЗАДАЧА]. Если вопрос еще актуален, могу предложить спокойный следующий шаг без давления: [СЛЕДУЮЩИЙ_ШАГ].

## Восстановление диалога после сумбурной переписки

[ИМЯ], давайте коротко соберу разговор по [БИЗНЕС], чтобы не потерять главное. Я вижу задачу так: [ЗАДАЧА]. Для ниши [НИША] логично начать с понятной структуры и первого этапа. Предлагаю следующий шаг: [СЛЕДУЮЩИЙ_ШАГ].

## Отправка структуры

[ИМЯ], отправляю предварительную структуру для [БИЗНЕС] под задачу [ЗАДАЧА]. Она показывает, как клиент будет проходить путь от знакомства с [НИША] до заявки. Посмотрите, пожалуйста, что нужно добавить или убрать. [СЛЕДУЮЩИЙ_ШАГ].

## Отправка КП

[ИМЯ], подготовил коммерческое предложение по [БИЗНЕС] с учетом задачи [ЗАДАЧА]. Внутри есть состав работ, этапы, сроки, стоимость и что потребуется от вашей стороны. После просмотра можем обсудить вопросы и согласовать [СЛЕДУЮЩИЙ_ШАГ].

## Follow-up через 2-3 дня

[ИМЯ], добрый день. Хотел уточнить, успели ли вы посмотреть информацию по [БИЗНЕС] и задаче [ЗАДАЧА]. Если удобно, можем коротко пройтись по вопросам и определить [СЛЕДУЮЩИЙ_ШАГ].

## Follow-up через 5-7 дней

[ИМЯ], возвращаюсь к разговору по [БИЗНЕС]. Если задача [ЗАДАЧА] пока не в приоритете, все нормально. Если актуально, могу предложить простой вариант продолжения для [НИША]: [СЛЕДУЮЩИЙ_ШАГ].

## Мягкое закрытие отказа

[ИМЯ], понял, спасибо за ответ. Тогда закрываю вопрос по [БИЗНЕС] и задаче [ЗАДАЧА] на текущий момент. Если позже понадобится понятная цифровая точка входа для клиентов в [НИША], можно будет вернуться к разговору. Хороший следующий шаг на будущее: [СЛЕДУЮЩИЙ_ШАГ].
