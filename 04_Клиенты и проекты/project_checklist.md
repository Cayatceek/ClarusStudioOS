# Чеклист проекта

## Назначение файла

Чеклист помогает вести проект по этапам и не пропускать важные договоренности, материалы, проверки и передачу результата клиенту.

## Когда использовать

- Перед стартом проекта.
- После согласования первого этапа.
- Во время разработки.
- Перед запуском.
- После передачи проекта клиенту.
- Перед оформлением кейса.

## Как использовать

Скопировать чеклист в папку клиента или проектный файл и отмечать пункты по мере выполнения. Если пункт не закрыт, проект нельзя считать полностью завершенным.

## 1. До старта

- [ ] Задача зафиксирована
- [ ] Понятно, что клиент хочет получить
- [ ] Понятно, какую проблему решает проект
- [ ] Создана карточка клиента
- [ ] В CRM обновлен статус
- [ ] Зафиксирован первый этап
- [ ] Записан следующий шаг

## 2. Согласование

- [ ] Первый этап согласован
- [ ] Стоимость согласована
- [ ] Порядок оплаты согласован
- [ ] Сроки согласованы
- [ ] Что входит зафиксировано
- [ ] Что не входит зафиксировано
- [ ] Возможный второй этап обозначен
- [ ] Клиент понимает рамки проекта

## 3. Материалы

- [ ] Список материалов отправлен клиенту
- [ ] Название бизнеса получено
- [ ] Описание услуг или товаров получено
- [ ] Фото или визуальные материалы получены
- [ ] Контакты получены
- [ ] Цены или условия получены
- [ ] Примеры работ получены
- [ ] Пожелания по стилю получены
- [ ] Недостающие материалы зафиксированы

## 4. Структура

- [ ] Структура готова
- [ ] Первый экран / стартовый сценарий продуман
- [ ] Путь клиента к заявке понятен
- [ ] Блоки идут в логичном порядке
- [ ] Кнопки и формы находятся в нужных местах
- [ ] Структура отправлена клиенту
- [ ] Структура согласована

## 5. Дизайн / визуальное направление

- [ ] Визуальные ориентиры собраны
- [ ] Цвета и настроение согласованы
- [ ] Основной экран подготовлен
- [ ] Клиент понимает визуальное направление
- [ ] Правки по визуалу зафиксированы
- [ ] Дизайн/визуальное направление согласовано

## 6. Разработка

- [ ] Разработка начата
- [ ] Основная структура собрана
- [ ] Контент размещен
- [ ] Форма или кнопка заявки добавлена
- [ ] Мобильная версия проверена
- [ ] Основные правки внесены
- [ ] Разработка завершена

## 7. Тестирование

- [ ] Все ссылки проверены
- [ ] Форма заявки проверена
- [ ] Контакты проверены
- [ ] Тексты проверены
- [ ] Мобильная версия проверена
- [ ] Ошибки исправлены
- [ ] Тестирование завершено

## 8. Запуск

- [ ] Проект готов к запуску
- [ ] Клиент подтвердил запуск
- [ ] Домен / ссылка / доступы проверены
- [ ] Проект запущен
- [ ] Финальная ссылка работает

## 9. Передача клиенту

- [ ] Клиент получил ссылку
- [ ] Клиент получил инструкцию
- [ ] Клиент понимает, как принимать заявки
- [ ] Клиент знает, что делать при вопросах
- [ ] Финальный результат зафиксирован в карточке клиента

## 10. После запуска

- [ ] Проверить работу через 1-2 дня
- [ ] Уточнить, есть ли вопросы у клиента
- [ ] Зафиксировать возможные доработки
- [ ] Отделить доработки от нового этапа
- [ ] Есть следующий шаг после запуска

## 11. Кейс

- [ ] Получен отзыв
- [ ] Разрешение на публикацию проверено
- [ ] Исходная ситуация описана
- [ ] Задача описана
- [ ] Решение описано
- [ ] Результат описан
- [ ] Кейс оформлен
- [ ] Проект перенесен в архив при необходимости

## Контрольные вопросы

- [ ] Зафиксирован ли первый этап?
- [ ] Понятно ли, что не входит?
- [ ] Есть ли материалы?
- [ ] Согласована ли стоимость?
- [ ] Есть ли следующий шаг после запуска?
