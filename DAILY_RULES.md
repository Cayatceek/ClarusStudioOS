# Ежедневные правила Clarus Studio

- Не писать клиенту без понятного следующего шага.
- Не обсуждать цену до понимания объема задачи.
- Не перегружать клиента техническими деталями.
- Каждый лид должен быть внесен в CRM-таблицу.
- Каждый проект должен идти по этапам.
- После каждого ответа клиента обновлять статус.
- Не обещать рост продаж.
- Продавать не "сайт", а понятную цифровую точку входа для заявки.
- Не действовать из эмоций.
- Перед отправкой сообщения проверить тон, цель и следующий шаг.
