# Ежедневные правила Clarus Studio

## Назначение файла

Этот файл открывается в начале рабочего дня и перед важными сообщениями клиентам. Его задача - удерживать работу студии в спокойном, управляемом режиме.

## Когда использовать

- Перед началом дня.
- Перед отправкой сообщения клиенту.
- Перед обсуждением цены.
- Перед стартом проекта.
- В конце дня перед закрытием задач.
- Когда появляется желание действовать резко, спешить или обещать больше, чем зафиксировано.

## Как использовать

Прочитать правила, выбрать активные действия на день и проверить контрольный чеклист. Если действие не проходит по правилам, сначала уточнить задачу, статус или следующий шаг.

## 10 главных правил дня

1. Не писать клиенту без понятного следующего шага.
2. Не обсуждать цену до понимания объема задачи.
3. Не перегружать клиента техническими деталями.
4. Каждый лид должен быть внесен в CRM-таблицу.
5. Каждый проект должен идти по этапам.
6. После каждого ответа клиента обновлять статус.
7. Не обещать рост продаж.
8. Продавать не "сайт", а понятную цифровую точку входа для заявки.
9. Не действовать из эмоций.
10. Перед отправкой сообщения проверить тон, цель и следующий шаг.

## Перед отправкой сообщения клиенту

Проверить:

- кому отправляется сообщение;
- почему сообщение нужно отправить сейчас;
- какая цель сообщения;
- какой следующий шаг предлагается;
- не звучит ли текст как давление;
- нет ли лишних технических деталей;
- обновлена ли CRM или карточка клиента.

Готовая внутренняя формулировка перед отправкой: "Я пишу клиенту, чтобы продвинуть конкретный следующий шаг, а не просто продолжить переписку".

## После ответа клиента

Сразу зафиксировать:

- дату последнего контакта;
- новый статус;
- краткий смысл ответа;
- следующий шаг;
- срок, если он есть;
- вопрос, который нужно уточнить.

Нельзя оставлять ответ клиента без обработки. Если клиент ответил, система должна измениться.

## Перед обсуждением цены

Цена обсуждается только после понимания:

- какую задачу нужно решить;
- что входит в первый этап;
- что не входит;
- какие материалы нужны от клиента;
- какой ожидаемый срок;
- какой следующий шаг после оплаты или согласования.

Если объем неясен, правильный ответ - не цена, а уточнение объема.

## Перед началом проекта

Проверить:

- создана ли карточка клиента;
- зафиксирован ли первый этап;
- согласована ли стоимость;
- согласован ли порядок оплаты;
- понятен ли список материалов;
- есть ли проектный чеклист;
- записаны ли риски и ограничения.

Проект не начинается "примерно". Проект начинается с зафиксированного первого этапа.

## В конце рабочего дня

Закрыть день короткой ревизией:

- какие сообщения отправлены;
- кто ответил;
- какие статусы изменились;
- какие проекты сдвинулись;
- какие оплаты ожидаются;
- что нужно сделать завтра.

Если результат дня не зафиксирован, на следующий день работа снова начнется с хаоса.

## Запрещённые действия

- Держать активного клиента только в переписке.
- Обещать срок без проверки объема.
- Называть стоимость без понимания первого этапа.
- Писать клиенту без цели.
- Давить на клиента follow-up сообщениями.
- Перегружать клиента внутренними техническими подробностями.
- Начинать работу без согласования оплаты.
- Удалять старые документы, которые связаны с решениями или договоренностями.

## Контрольный чеклист на 5 минут

- [ ] У каждого активного лида есть статус
- [ ] У каждого активного лида есть следующий шаг
- [ ] Нет клиентов "в голове"
- [ ] Нет обещаний без фиксации
- [ ] Нет цены без понятного объема
- [ ] Нет проекта без первого этапа
- [ ] Нет сообщения без цели
