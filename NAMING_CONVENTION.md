# Правила названия файлов

Единые названия помогают быстро находить документы, понимать дату создания и не путать версии.

## Общие правила

- В технических файлах не использовать пробелы.
- Для публичных файлов использовать английский транслит.
- Для внутренних документов можно использовать русский язык.
- Дату ставить в начало названия, если документ связан с клиентом, проектом, КП, кейсом или заметкой.
- Название должно быть коротким и понятным без лишних слов.

## Форматы

- Документы: `YYYY-MM-DD_short-topic_type.md`
- КП: `YYYY-MM-DD_client_commercial-offer.docx`
- Кейсы: `YYYY-MM-DD_client_case-study.md`
- Заметки по клиенту: `YYYY-MM-DD_client_notes.md`

## Примеры

- `2026-01-10_catalog-structure_notes.md`
- `2026-01-12_business-placeholder_commercial-offer.docx`
- `2026-01-20_business-placeholder_case-study.md`
- `2026-01-25_business-placeholder_notes.md`

## Версии

Если нужно сохранить несколько версий, добавлять короткую пометку:

- `v1`
- `v2`
- `final`
- `old-version`

Пример: `2026-01-12_business-placeholder_commercial-offer_v2.docx`
